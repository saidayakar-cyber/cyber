LogWatch Project Documentation

1. Project Overview

The LogWatch project is a Windows-based log monitoring solution designed to detect and track important security-related events in the system. It focuses on failed login attempts, user account creation and deletion, group membership modifications, and privilege use events. This helps administrators identify unauthorized access attempts and account changes.

2. How It Works

LogWatch collects data from the Windows Security Event Log. The PowerShell script filters for specific Event IDs (4625, 4720, 4726, 4728, 4729, 4672). It outputs alerts in JSON format to C:\LogWatch\alerts.json, and can also display results in the console.

It runs automatically via Windows Task Scheduler every 1–2 minutes, ensuring continuous monitoring.

3. Implementation Steps

Created project folder: C:\LogWatch

Developed PowerShell script (LogWatch.ps1) to query and filter Security Events

Tested manually by triggering failed logons and creating/deleting users

Configured Task Scheduler to run script every minute with PowerShell

4. Sample Output

Example alerts generated and saved in alerts.json:

- Rule: FailedLogon (ID: 4625, Severity: High) Message: An account failed to log on... Failure Reason: An Error occurred during Logon. Status: 0xC000006D

- Rule: UserAdded (ID: 4720, Severity: High) Message: A user account was created... Account Name: test1

- Rule: UserDeleted (ID: 4726, Severity: High) Message: A user account was deleted... Account Name: test1

- Rule: PrivilegeUse (ID: 4672, Severity: Medium) Message: Special privileges assigned... SeDebugPrivilege SeBackupPrivilege

5. Benefits

• Detects brute-force login attempts • Tracks creation and deletion of accounts • Monitors group membership changes • Identifies logons with elevated rights • Stores data in JSON format usable in SIEM tools

8. Project Update: Real-Time Email Alert Integration

An important improvement made to the LogWatch project is the addition of real-time email alerts. Instead of only storing alerts in alerts.json or displaying them on the console, the system now sends email notifications directly to administrators using Gmail. This enhancement ensures that critical security events are not missed and administrators can respond faster.

This command executes LogWatch to check events from the last 5 minutes and outputs results to the console. With the email integration enabled, the same execution also triggers a real-time email alert to the configured Gmail account.

To test this feature, run the following command in PowerShell:

.\LogWatch.ps1 -SinceMinutes 5 -ToConsole
