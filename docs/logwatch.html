LogWatch — Windows Security Log Monitoring

Detects and summarizes key Windows Security events to surface suspicious activity quickly.

Scope

  • Failed logons (4625)
  • User create/delete (4720 / 4726)
  • Group membership add/remove (4728 / 4729)
  • Special privileges use (4672)

How It Works

A PowerShell script queries the Security log for target Event IDs in a time window and serializes findings to JSON at C:\LogWatch\alerts.json. Optionally echoes to console.

Setup

  1. Create folder C:\LogWatch
  2. Place LogWatch.ps1 there
  3. Test with known events (failed login, user add/delete)
  4. Schedule with Task Scheduler every 1–2 minutes

Sample Output

Rule: FailedLogon (4625, High)
Message: An account failed to log on... Status: 0xC000006D

Email Alerts (Optional)

Run .\LogWatch.ps1 -SinceMinutes 5 -ToConsole during testing to simulate recent events and verify email delivery rules.

Outcomes

  • Faster initial triage
  • Account change visibility
  • JSON output ready for SIEM
