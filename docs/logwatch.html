LogWatch — Windows Security Log Monitoring

LogWatch detects and summarizes important Windows Security events to surface credential attacks and risky account changes quickly.

1. Project Overview

Windows-based monitoring focused on Event IDs 4625 (failed logon), 4720/4726 (user create/delete), 4728/4729 (group changes), and 4672 (special privileges). Alerts are written to JSON for SIEM ingestion and can be echoed to console.

2. How It Works

A PowerShell script queries the Security Event Log for the target IDs within a time window and formats each match into JSON with rule name, ID, severity, and key fields. Results are saved to C:\LogWatch\alerts.json and optionally printed to console. A Windows Task Scheduler job runs the script every 1–2 minutes for near‑real‑time visibility.

3. Implementation Steps

  1. Create the project folder: C:\LogWatch
  2. Place LogWatch.ps1 in that folder
  3. Test by triggering a failed logon and a test user add/delete
  4. Configure Task Scheduler to run powershell.exe -File C:\LogWatch\LogWatch.ps1 on a 1–2 minute schedule

4. Sample Output

Rule: FailedLogon (4625, High)
Message: An account failed to log on... Status: 0xC000006D

Rule: UserAdded (4720, High)
Message: A user account was created... Account Name: test1

Rule: UserDeleted (4726, High)
Message: A user account was deleted... Account Name: test1

Rule: PrivilegeUse (4672, Medium)
Message: Special privileges assigned... SeDebugPrivilege SeBackupPrivilege

5. Benefits

  • Detects brute‑force and password‑spray attempts
  • Tracks account creation/deletion and group membership changes
  • Highlights logons with elevated rights
  • Produces JSON that can be shipped to a SIEM

6. Real‑Time Email Alerts (Improvement)

With email integration, critical alerts are delivered to administrators immediately while still being written to JSON.

Test: run .\LogWatch.ps1 -SinceMinutes 5 -ToConsole, which checks recent events and (when configured) triggers email notifications.

7. Future Enhancements

  • Include successful logon tracking (4624) for correlation
  • Log rotation/archiving policies
  • Alternate alert transports (Teams/Slack webhooks)

8. Conclusion

LogWatch evolved from a basic event query into a practical monitoring aid with JSON output and optional real‑time alerting—reducing time to detection and centralizing evidence for incident response.
