Python Password Manager

Local, file‑backed vault with hashed storage and clipboard timeout.

Design

  • Entries keyed by service + login
  • Fields: URL, email/username, secret
  • Secrets hashed (bcrypt/PBKDF2)

CLI

python pm.py add -s github -u https://github.com -e you@example.com -l mygithubuser
python pm.py get -s github -l mygithubuser --clip-timeout 10
python pm.py list

Implementation Notes

  • Validate inputs and avoid overwrites
  • Don’t log secrets
  • Unit test core crypto and clipboard code

Outcomes

  • No plaintext storage
  • Reduced copy‑paste errors

Roles & Responsibilities

  • Defined CLI flows for adding, retrieving, and listing passwords with a single master credential.
  • Implemented hashed storage using a standard library KDF (e.g., PBKDF2/Argon2/Bcrypt) and organized file‑backed data.
  • Added a safe clipboard timeout to lower the risk of secret exposure after copy.
  • Wrote setup & run instructions and validated the tool on multiple test entries.
  • Documented edge cases (bad input, missing files) with graceful error handling.

What I Gained

  • Practical understanding of master‑password vault design and key‑derivation best practices at a beginner level.
  • Improved Python fundamentals: file I/O, structured JSON/CSV storage, argument parsing, and exceptions.
  • Security hygiene habits around secrets handling and minimal data exposure in a local tool.
  • Experience writing concise CLI UX and documentation other learners can follow.
