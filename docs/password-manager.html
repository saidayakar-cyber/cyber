Python Password Manager

Local, file‑backed vault with hashed storage and clipboard timeout.

Design

  • Entries keyed by service + login
  • Fields: URL, email/username, secret
  • Secrets hashed (bcrypt/PBKDF2)

CLI

python pm.py add -s github -u https://github.com -e you@example.com -l mygithubuser
python pm.py get -s github -l mygithubuser --clip-timeout 10
python pm.py list

Implementation Notes

  • Validate inputs and avoid overwrites
  • Don’t log secrets
  • Unit test core crypto and clipboard code

Outcomes

  • No plaintext storage
  • Reduced copy‑paste errors

Roles & Responsibilities

  • Designed CLI flows for add, get, and list credentials protected by a master password.
  • Implemented hashed storage using a standard KDF (e.g., PBKDF2/Argon2/Bcrypt) and file‑backed data layout.
  • Added a safe clipboard timeout to minimize secret exposure after copy operations.
  • Wrote clear Setup & Run steps and validated with sample entries.
  • Handled edge cases with helpful error messages (bad input, missing files).

What I Gained

  • Practical understanding of password‑vault fundamentals and key‑derivation best practices.
  • Improved Python skills: structured file I/O, simple data models, and exception handling.
  • Better security hygiene around secrets handling and local storage.
  • Experience documenting a small security tool for other beginners to use.
