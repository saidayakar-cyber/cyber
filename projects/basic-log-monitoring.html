Basic Log Monitoring

Collected Windows/Linux security logs and built a simple triage checklist to quickly identify failed logons, privilege events, and added users.

  • Environment & Tools
  • How I built it
  • How to run

  • Repository

Environment & Tools

Windows 10/11 & Ubuntu; Event Viewer / journalctl; simple syslog forwarder

How I built it

  1. Defined target event IDs and journald filters
  2. Exported recent events and normalized fields
  3. Documented a triage checklist for incident responders

How to run

  1. Open Event Viewer → Security log
  2. journalctl -u ssh --since '2 hours ago'

Outcomes

  • Reduced initial triage time from ~10 mins to ~4 mins in lab tests
  • Created a repeatable, step-by-step checklist for faster validation
